## 🎯 Tabel Referensi Cepat

### Operator Logika Dasar

| Operator | Symbol | Nama | Truth Table | Contoh Code |
|----------|--------|------|-------------|-------------|
| AND | ∧ | Konjungsi | T only if both T | `a && b` |
| OR | ∨ | Disjungsi | F only if both F | `a \|\| b` |
| NOT | ¬ | Negasi | Inverts value | `!a` |
| IMPLIES | → | Implikasi | F only if T→F | `!a \|\| b` |
| IFF | ↔ | Ekuivalensi | T if same value | `a == b` |
| NAND | ↑ | Not-And | F only if both T | `!(a && b)` |
| NOR | ↓ | Not-Or | T only if both F | `!(a \|\| b)` |
| XOR | ⊕ | Exclusive-Or | T if different | `a ^ b` |

### Hukum-Hukum Logika Penting

| Hukum | Formula |
|-------|---------|
| **Identity** | `P ∧ T ≡ P` <br> `P ∨ F ≡ P` |
| **Domination** | `P ∨ T ≡ T` <br> `P ∧ F ≡ F` |
| **Idempotent** | `P ∧ P ≡ P` <br> `P ∨ P ≡ P` |
| **Double Negation** | `¬¬P ≡ P` |
| **Commutative** | `P ∧ Q ≡ Q ∧ P` <br> `P ∨ Q ≡ Q ∨ P` |
| **Associative** | `(P ∧ Q) ∧ R ≡ P ∧ (Q ∧ R)` |
| **Distributive** | `P ∧ (Q ∨ R) ≡ (P ∧ Q) ∨ (P ∧ R)` |
| **De Morgan** | `¬(P ∧ Q) ≡ ¬P ∨ ¬Q` <br> `¬(P ∨ Q) ≡ ¬P ∧ ¬Q` |
| **Implication** | `P → Q ≡ ¬P ∨ Q` |
| **Contrapositive** | `P → Q ≡ ¬Q → ¬P` |

### Kuantor Logika Predikat

| Kuantor | Symbol | Dibaca | Contoh |
|---------|--------|--------|--------|
| **Universal** | ∀ | "Untuk semua" | `∀x P(x)` - Semua x memenuhi P |
| **Existential** | ∃ | "Ada" | `∃x P(x)` - Ada x yang memenuhi P |

**Negasi Kuantor:**
- `¬∀x P(x) ≡ ∃x ¬P(x)` - "Tidak semua" = "Ada yang tidak"
- `¬∃x P(x) ≡ ∀x ¬P(x)` - "Tidak ada" = "Semua tidak"

## 📚 BAGIAN I: LOGIKA PROPOSISIONAL

<details>
<summary><b>BAB 1: Pengenalan Logika Matematika</b></summary>

### 1.1 Pendahuluan

Logika matematika adalah studi formal tentang **penalaran yang benar** (valid reasoning). Dalam ilmu komputer, logika matematika menjadi fondasi untuk:
- 🔧 Desain algoritma
- ✅ Verifikasi program
- 📝 Desain bahasa pemrograman
- 🤖 Kecerdasan buatan
- 💾 Database query

### 1.2 Argumen

**Definisi**: Argumen adalah sekumpulan pernyataan yang terdiri dari **premis-premis** dan **kesimpulan**.

**Struktur Argumen**:
```
Premis 1: P₁
Premis 2: P₂
...
Premis n: Pₙ
─────────────
Kesimpulan: C
```

**Contoh Argumen Valid**:
```
Premis 1: Semua manusia mortal
Premis 2: Socrates adalah manusia
─────────────────────────────────
Kesimpulan: Socrates mortal ✓
```

### 1.3 Validitas Argumen

Argumen dikatakan **valid** jika:
> Ketika semua premis bernilai benar, maka kesimpulan **pasti** benar

⚠️ **Penting**: 
- Argumen Valid ≠ Kesimpulan Benar
- Argumen valid hanya menjamin: `premis benar → kesimpulan benar`
- Argumen bisa valid tapi kesimpulannya salah (jika premisnya salah)

**Contoh Argumen Valid dengan Kesimpulan Salah**:
```
Premis 1: Semua burung bisa terbang  (SALAH - pinguin tidak bisa)
Premis 2: Pinguin adalah burung      (BENAR)
────────────────────────────────────
Kesimpulan: Pinguin bisa terbang     (SALAH)
```
Argumen ini VALID (struktur benar) tapi kesimpulan SALAH (karena premis 1 salah).

### 1.4 Logika Klasik

Karakteristik logika klasik:
- **Bivalensi**: Setiap proposisi bernilai `True` ATAU `False` (tidak ada nilai tengah)
- **Law of Excluded Middle**: `P ∨ ¬P` selalu benar
- **Law of Non-contradiction**: `¬(P ∧ ¬P)` selalu benar

### 1.5 Logika Modern

Perkembangan dari logika klasik dengan:
- Formalisasi yang lebih ketat
- Sistem aksiomatik
- Kalkulus proposisional dan predikat
- Teori model dan bukti

### 1.6 Logika Banyak Nilai

Sistem logika dengan **lebih dari dua** nilai kebenaran:

| Jenis | Nilai | Aplikasi |
|-------|-------|----------|
| **Three-valued** | True, False, Unknown | Database (NULL values) |
| **Fuzzy Logic** | [0, 1] (continuous) | AI, Control Systems |

**Contoh dalam SQL**:
```sql
SELECT * FROM users WHERE email IS NULL  -- Unknown value!
```

### 1.7 Mengenal Logika Lebih Dalam

Eksplorasi sistem logika lainnya:
- 🔀 **Modal logic** - Necessity & Possibility
- ⏰ **Temporal logic** - Time-based reasoning
- 🏗️ **Intuitionistic logic** - Constructive mathematics

<summary><b>BAB 2: Pengantar Logika Proposisional</b></summary>

### 2.1 Pendahuluan

**Proposisi** adalah pernyataan yang memiliki nilai kebenaran (benar atau salah).

**Contoh Proposisi**:
- ✅ "2 + 2 = 4" (proposisi, benar)
- ✅ "Jakarta adalah ibukota Indonesia" (proposisi, benar)
- ✅ "10 < 5" (proposisi, salah)
- ❌ "Tutup pintu!" (bukan proposisi - perintah)
- ❌ "Apakah hujan?" (bukan proposisi - pertanyaan)
- ❌ "x > 5" (bukan proposisi - bergantung pada x)

**Dalam Pemrograman**:
```python
# Proposisi (nilai kebenaran jelas)
is_valid = (age >= 18)  # True atau False

# Bukan proposisi
name = input("Nama: ")  # Tidak punya nilai kebenaran
```

### 2.2 Argumen-argumen

Berbagai bentuk argumen valid dalam logika proposisional:

#### 1. Modus Ponens
```
P → Q
P
─────
∴ Q
```

**Contoh**:
```
Jika hujan, maka jalan basah
Hujan
────────────────────────────
Jadi, jalan basah
```

**Dalam Code**:
```python
if is_raining:
    road_is_wet = True
# Jika is_raining == True, maka road_is_wet == True
```

#### 2. Modus Tollens
```
P → Q
¬Q
─────
∴ ¬P
```

**Contoh**:
```
Jika hujan, maka jalan basah
Jalan tidak basah
────────────────────────────
Jadi, tidak hujan
```

#### 3. Silogisme Hipotesis
```
P → Q
Q → R
─────
∴ P → R
```

**Contoh**:
```
Jika belajar, maka lulus
Jika lulus, maka bahagia
────────────────────────
Jadi, jika belajar, maka bahagia
```

#### 4. Silogisme Disjungtif
```
P ∨ Q
¬P
─────
∴ Q
```

**Contoh**:
```
Hari ini cerah atau hujan
Hari ini tidak cerah
──────────────────────
Jadi, hari ini hujan
```

### 2.3 Proposisi-proposisi

Jenis-jenis proposisi:

| Jenis | Definisi | Contoh |
|-------|----------|--------|
| **Atomik** | Tidak dapat dipecah | "Hujan", "P", "Q" |
| **Majemuk** | Kombinasi proposisi atomik | "P ∧ Q", "P → Q" |

### 2.4 Pemberian Nilai

**Assignment** (penugasan nilai): Memberikan nilai `True` atau `False` pada variabel proposisi.

**Contoh**:
```
P = True
Q = False
R = True

Maka: P ∧ Q = True ∧ False = False
```

**Dalam Python**:
```python
P = True
Q = False
R = True

result = P and (Q or R)  # True and (False or True) = True
print(result)  # Output: True
```

<summary><b>BAB 3: Tabel Kebenaran</b></summary>

### 3.1 Pendahuluan

**Tabel kebenaran** adalah tabel yang menunjukkan nilai kebenaran dari proposisi majemuk untuk **semua kemungkinan** nilai proposisi atomiknya.

**Jumlah baris**: Untuk `n` variabel proposisi → `2ⁿ` baris

### 3.2 Tabel Kebenaran

**Contoh** (2 variabel):
```
P | Q | (operasi)
--|---|----------
T | T | ...
T | F | ...
F | T | ...
F | F | ...
```

### 3.3 Perangkai Logika atau Operator

#### 3.3.1 Konjungsi (AND / ∧)

`P ∧ Q` benar **hanya jika** P dan Q **keduanya benar**.

| P | Q | P ∧ Q |
|---|---|-------|
| T | T | **T** |
| T | F | F |
| F | T | F |
| F | F | F |

**Aplikasi**:
```python
# Python
if age >= 18 and has_id:
    print("Boleh masuk")
```

```javascript
// JavaScript
if (age >= 18 && hasId) {
    console.log("Boleh masuk");
}
```

**Hardware**: Gerbang AND
```
  P ──┐
      ├─AND──→ Q
  Q ──┘
```

#### 3.3.2 Disjungsi (OR / ∨)

`P ∨ Q` benar jika **setidaknya satu** dari P atau Q benar (inclusive OR).

| P | Q | P ∨ Q |
|---|---|-------|
| T | T | **T** |
| T | F | **T** |
| F | T | **T** |
| F | F | F |

**Aplikasi**:
```python
# Python
if is_admin or is_moderator:
    print("Akses granted")
```

#### 3.3.3 Negasi (NOT / ¬)

`¬P` adalah **kebalikan** dari P.

| P | ¬P |
|---|----|
| T | F  |
| F | T  |

**Aplikasi**:
```python
# Python
if not is_logged_in:
    redirect_to_login()
```

#### 3.3.4 Implikasi (→)

`P → Q` dibaca "Jika P maka Q". Benar **kecuali** P benar dan Q salah.

| P | Q | P → Q |
|---|---|-------|
| T | T | **T** |
| T | F | **F** |
| F | T | **T** |
| F | F | **T** |

⚠️ **Penting**: "False implies anything" - Jika premis salah, implikasi selalu benar!

**Ekuivalen**: `P → Q ≡ ¬P ∨ Q`

**Aplikasi**:
```python
# P → Q sama dengan ¬P ∨ Q
# if P then Q  ===  (not P) or Q

def check_valid(age):
    # Jika age < 18, maka not_allowed
    # age < 18 → not_allowed
    return (age >= 18) or not_allowed  # ¬P ∨ Q
```

#### 3.3.5 Ekuivalensi (↔)

`P ↔ Q` dibaca "P jika dan hanya jika Q". Benar jika P dan Q **memiliki nilai yang sama**.

| P | Q | P ↔ Q |
|---|---|-------|
| T | T | **T** |
| T | F | F |
| F | T | F |
| F | F | **T** |

**Ekuivalen**: `P ↔ Q ≡ (P → Q) ∧ (Q → P)`

**Aplikasi**:
```python
# Checking equivalence
result = (P == Q)  # True jika sama
```

### 3.4 Perangkai Logika Lainnya

#### 3.4.1 NAND (Not-And / ↑ / |)

`P ↑ Q` adalah negasi dari konjungsi.

| P | Q | P ↑ Q |
|---|---|-------|
| T | T | **F** |
| T | F | **T** |
| F | T | **T** |
| F | F | **T** |

**Formula**: `P ↑ Q ≡ ¬(P ∧ Q)`

🌟 **Penting**: NAND adalah **universal gate** - Semua gerbang logika bisa dibuat dari NAND!

**Contoh**:
```
NOT P  = P NAND P
P AND Q = (P NAND Q) NAND (P NAND Q)
P OR Q  = (P NAND P) NAND (Q NAND Q)
```

#### 3.4.2 NOR (Not-Or / ↓)

`P ↓ Q` adalah negasi dari disjungsi.

| P | Q | P ↓ Q |
|---|---|-------|
| T | T | **F** |
| T | F | F |
| F | T | F |
| F | F | **T** |

**Formula**: `P ↓ Q ≡ ¬(P ∨ Q)`

🌟 **Penting**: NOR juga adalah **universal gate**!

#### 3.4.3 XOR (Exclusive-Or / ⊕)

`P ⊕ Q` benar jika P dan Q **memiliki nilai berbeda**.

| P | Q | P ⊕ Q |
|---|---|-------|
| T | T | **F** |
| T | F | **T** |
| F | T | **T** |
| F | F | **F** |

**Formula**: `P ⊕ Q ≡ (P ∨ Q) ∧ ¬(P ∧ Q)`

**Aplikasi**:

1. **Enkripsi Sederhana** (XOR Cipher):
```python
def xor_encrypt(text, key):
    return ''.join(chr(ord(c) ^ key) for c in text)

def xor_decrypt(encrypted, key):
    return ''.join(chr(ord(c) ^ key) for c in encrypted)

message = "HELLO"
key = 123
encrypted = xor_encrypt(message, key)
decrypted = xor_decrypt(encrypted, key)  # Back to "HELLO"
```

2. **Swap tanpa Variable Tambahan**:
```python
# Swap a dan b menggunakan XOR
a = a ^ b
b = a ^ b  # b = (a ^ b) ^ b = a
a = a ^ b  # a = (a ^ b) ^ a = b
```

3. **Parity Checking**:
```python
def even_parity(bits):
    # XOR semua bits, hasilnya harus 0 untuk even parity
    result = 0
    for bit in bits:
        result ^= bit
    return result == 0
```

## 💡 Aplikasi dalam Ilmu Komputer

### 1. 🖥️ Programming Languages

**Type Systems** (Curry-Howard Correspondence):
```
Logic          ↔  Programming
─────────────────────────────
P → Q          ↔  Function type: P → Q
P ∧ Q          ↔  Product type (tuple): (P, Q)
P ∨ Q          ↔  Sum type (union): P | Q
⊥ (falsum)     ↔  Empty type (never)
Proof          ↔  Program
Proof checking ↔  Type checking
```

**Contoh dalam TypeScript**:
```typescript
// P → Q: Function type
type Implies<P, Q> = (p: P) => Q;

// P ∧ Q: Product type (tuple)
type And<P, Q> = [P, Q];

// P ∨ Q: Sum type (union)
type Or<P, Q> = P | Q;
```

### 2. ✅ Formal Verification

**SAT Solvers** - Menyelesaikan Boolean satisfiability problem:
```python
# Example: Sudoku solving dengan SAT
# Setiap cell (i,j) dengan value v: x[i][j][v]
# Constraint: Setiap cell hanya satu nilai
for i in range(9):
    for j in range(9):
        # x[i][j][1] ∨ x[i][j][2] ∨ ... ∨ x[i][j][9] (at least one)
        # ¬(x[i][j][v] ∧ x[i][j][w]) for v ≠ w (at most one)
```

**Model Checkers** - Verifikasi sistem:
```
Properties: ∀ states: safety_property(state)
Check: Apakah property selalu terpenuhi?
```

### 3. 💾 Databases

**SQL Queries** menggunakan logika:
```sql
-- Logika: ∃u (User(u) ∧ age(u) > 18 ∧ verified(u))
SELECT * FROM users 
WHERE age > 18 AND verified = true;

-- Logika: ∀p (Product(p) → (p.stock > 0 ∨ p.backorder))
SELECT * FROM products 
WHERE stock > 0 OR backorder = true;
```

**Query Optimization** menggunakan hukum Boolean:
```sql
-- Original: NOT (active = 0 AND verified = 0)
-- De Morgan: (active = 1 OR verified = 1)
SELECT * FROM users WHERE active = 1 OR verified = 1;
```

### 4. 🤖 Artificial Intelligence

**Expert Systems** - Knowledge representation:
```prolog
% Prolog - Logic Programming
human(socrates).
mortal(X) :- human(X).

?- mortal(socrates).
% Output: true
```

**Planning** (STRIPS):
```
State: At(Home) ∧ ¬Has(Food)
Goal: At(Home) ∧ Has(Food)
Actions: Go(Store), Buy(Food), Go(Home)
```

### 5. 🔐 Computer Security

**Access Control** dengan logika:
```python
def can_access(user, resource):
    # (Admin ∨ Owner) ∧ ¬Banned ∧ (Public ∨ HasPermission)
    return ((user.is_admin or user.is_owner(resource)) and
            not user.is_banned and
            (resource.is_public or user.has_permission(resource)))
```

**Cryptography** - XOR dalam stream cipher:
```python
def stream_cipher(plaintext, key_stream):
    # C = P ⊕ K (encrypt)
    # P = C ⊕ K (decrypt)
    return bytes(p ^ k for p, k in zip(plaintext, key_stream))
```

### 6. ⚙️ Hardware Design

**Logic Gates** - Implementasi rangkaian:
```
Half Adder:
  Sum = A ⊕ B
  Carry = A ∧ B

Full Adder:
  Sum = A ⊕ B ⊕ Cin
  Cout = (A ∧ B) ∨ (Cin ∧ (A ⊕ B))
```

**Boolean Simplification** - Optimisasi hardware:
```
Original: (A ∧ B) ∨ (A ∧ ¬B)
Simplified: A  [Using distributive + complement]
Benefit: Fewer gates → Lower cost, faster, less power
```

### Contoh Penggunaan Z3 (Python)

```python
from z3 import *

# Define variables
p, q, r = Bools('p q r')

# Create solver
solver = Solver()

# Add constraints: (p ∨ q) ∧ (¬p ∨ r) ∧ ¬r
solver.add(Or(p, q))
solver.add(Or(Not(p), r))
solver.add(Not(r))

# Check satisfiability
if solver.check() == sat:
    model = solver.model()
    print(f"SAT! Model: p={model[p]}, q={model[q]}, r={model[r]}")
else:
    print("UNSAT!")
```
